# 대학수학 교과서 해설 사이트 — 설계 문서

날짜: 2026-07-02
상태: 승인됨 (사용자: "어차피 다 할거니까 시작해")

## 1. 목적

유명 대학수학 교과서를 기준으로 공부하면서, 이해한 내용을 **"진짜 이해하기 쉬운"
해설**로 정리하는 정적 웹사이트. 임용시험 통과가 주 목표는 아니고 최종 점검
수단 — 따라서 임용 기출문제를 챕터별로 매핑해서 제공한다.

첫 과목: **해석학** (Rudin, *Principles of Mathematical Analysis*, 3판 기준).
이후 선형대수(Friedberg), 현대대수(Fraleigh), 미분기하(do Carmo), 정수론(Burton),
위상수학(Munkres), 복소해석(Brown & Churchill), 확률통계(Hogg) 등으로 확장.

### 핵심 가치 (우선순위 순)

1. **이해하기 쉬운 설명** — 엄밀한 정의를 보여주기 *전에* 일상 언어 직관,
   동기("왜 이걸 정의하나"), 예시/반례를 먼저 제시한다.
2. **정의 상호 링크** — 본문에서 등장하는 모든 용어는 그 정의로 링크된다.
   호버(모바일: 탭)하면 짧은 정의 툴팁이 뜨고, 클릭하면 정의 위치로 이동한다.
   "기억 안 나는 정의 때문에 책을 뒤지는" 마찰을 없애는 것이 목적.
3. **문제 제공** — 챕터마다 (a) 개념 확인 문제(자체 제작), (b) 교과서
   연습문제(재서술 + 힌트/풀이), (c) 한국 중등임용 기출(연도·문항 표기,
   관련 개념 태그) 3종을 제공한다.

### 비목표 (당장 하지 않는 것)

- 프레임워크·빌드 도구 사용 (사용자 명시 요구: 순수 HTML/CSS/JS)
- 서버 기능, 계정, 진도 추적, 댓글
- 검색 기능 (콘텐츠가 쌓이면 클라이언트 사이드 검색을 나중에 추가 가능)

## 2. 기술 스택

- **HTML + CSS + 바닐라 JS. 빌드 없음.** 파일을 그대로 서빙한다.
- **KaTeX 로컬 번들** (`assets/vendor/katex/`) + auto-render 확장.
  CDN이 아닌 로컬 번들인 이유: 오프라인에서도 동작, 라즈베리파이 로컬 서빙에
  적합, 외부 의존성 제거.
- 로컬 확인: `python3 -m http.server` (file:// 로는 JS 모듈/fetch 제약이 있어
  서버 권장).

## 3. 디렉토리 구조

```

│   ├── site.js                 # 공통 레이아웃 주입, KaTeX 초기화, 용어 툴팁
```

## 4. 핵심 시스템 설계

### 4.1 목차 시스템 (`data/toc.js`)

전역 `window.MATH_TOC` 객체 하나. 과목별로 `{ slug, title, book, chapters: [{file, num, title}] }`.
`site.js`가 이 데이터로 다음을 자동 생성한다:

- 홈의 과목 카드 목록
- 과목 index의 챕터 목록
- 각 챕터 페이지의 사이드바(현재 과목 챕터 목록), 상단 브레드크럼, 하단 이전/다음 링크

**새 챕터 추가 = HTML 파일 1개 + toc.js 한 줄.** 네비게이션 수작업 없음.

### 4.2 용어 상호 링크 시스템 (`data/terms.js` + `site.js`)

- 전역 `window.MATH_TERMS` 객체: `slug → { ko, en, short, href }`.
  `short`는 툴팁용 1–3문장 요약 정의(KaTeX 수식 허용). `href`는 정식 정의
  위치(`페이지#def-slug`).
- 정의 박스는 `id="def-<slug>"` 앵커를 가진다.
- 본문 참조는 `<a class="term" data-term="<slug>">거리공간</a>` 형태로 쓴다.
  `site.js`가 로드 시 href를 채우고 호버/탭 툴팁(짧은 정의 + "정의 보기" 링크)을
  단다. 등록 안 된 slug는 콘솔 경고 + 시각적 표시(개발 중 발견 용이).
- 링크 규칙: 자기 정의 박스 안을 제외하고 **아낌없이 링크**한다. 독자가 어느
  지점에서 읽기 시작해도 막히지 않게.

### 4.3 콘텐츠 블록 (style.css의 표준 클래스)

| 클래스 | 용도 |
|---|---|
| `.box-intuition` | 💡 "쉽게 말하면" — 정의·정리 앞에 오는 일상 언어 설명 |
| `.box-def` | 📘 엄밀한 정의 (한/영 용어 병기, `id="def-*"` 앵커) |
| `.box-thm` | 📐 정리/명제/보조정리 — "이 정리가 하는 일" 요약 문장 필수 |
| `.box-warn` | ⚠️ 흔한 오해 / 함정 |
| `.box-example` | ✏️ 예제 (풀이 포함) |
| `details.proof` | 접을 수 있는 증명 — 열기 전 "증명 전략" 한 줄 요약 표시 |
| `.problem` | 문제 카드 — 출처 배지 + `<details>` 힌트 → `<details>` 풀이 |

### 4.4 챕터 페이지 표준 구성 ("이해하기 쉽게"의 구체화)

1. **도입** — 이 챕터를 왜 배우는지, 어떤 질문에 답하는지 일상 언어로.
2. **본문 섹션들** — 각 개념마다 고정 순서:
   직관(💡) → 엄밀한 정의(📘) → 예시와 반례(✏️) → 흔한 오해(⚠️).
   정리는 "무엇을 해주는 정리인지" 먼저, 증명은 접어서.
   그림이 도움이 되는 곳엔 인라인 SVG.
3. **한 장 요약** — 챕터 전체를 표/다이어그램 하나로.
4. **연습문제** — 개념 확인(쉬움) → 교과서 문제(재서술) → 임용 기출.
   모든 문제에 힌트와 풀이를 단계적 `<details>`로.

### 4.5 문제 저작권 처리

- Rudin 연습문제: 원문 전재 대신 **재서술**하고 "Rudin 1장 연습 5 기반"처럼 출처 표기.
- 임용 기출: 국가 시험 문제(공개 자료). 연도·문항 번호 표기, 필요시 표기법만 사이트 표준에 맞게 손질.

## 5. 해석학 챕터 로드맵 (Rudin PMA 기준)

| # | 파일 | 주제 | 임용 관련 핵심 |
|---|---|---|---|
| 1 | ch01 | 실수와 복소수 체계 | 상한/하한, 완비성 공리, 아르키메데스 성질 |
| 2 | ch02 | 기초 위상 | 열린/닫힌 집합, 컴팩트, 연결, 완전집합 |
| 3 | ch03 | 수열과 급수 | 수렴 판정, 상극한/하극한, 코시 수열 |
| 4 | ch04 | 연속 | 연속과 컴팩트/연결, 균등연속 |
| 5 | ch05 | 미분 | 평균값 정리, 로피탈, 테일러 |
| 6 | ch06 | 리만-스틸체스 적분 | 적분 가능 조건, 미적분학의 기본정리 |
| 7 | ch07 | 함수열과 함수급수 | 균등수렴과 연속/미분/적분의 교환 |
| 8 | ch08 | 특수 함수 | 멱급수, 지수/로그/삼각함수, 푸리에 급수 |

## 6. 검증

- `tools/check_site.py`: 전 페이지를 파싱해 (a) 내부 링크·앵커 유효성,
  (b) `data-term` 슬러그가 terms.js에 존재하는지, (c) toc.js와 실제 파일
  일치 여부를 검사. 콘텐츠 추가 때마다 실행.
- 로컬 서버 + 브라우저(또는 curl)로 KaTeX 렌더링, 툴팁, 네비게이션 동작 확인.
- 수학적 정확성: 콘텐츠 작성 시 적대적 검증(오류 찾기 전담 리뷰) 통과 후 게시.

## 7. 마일스톤

- **M1 — 뼈대**: 공통 chrome(스타일·레이아웃·툴팁·네비), 홈, 해석학 과목 index, KaTeX 번들, 검사 스크립트.
- **M2 — 해석학 1장**: 위 템플릿의 완전한 첫 사례. 용어 등록 + 문제 3종 포함.
